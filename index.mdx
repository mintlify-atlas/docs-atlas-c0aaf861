Lucy is a takeover-first environment manager for Minecraft servers. Instead of requiring a blank slate, Lucy inspects the server you already run, adopts it into a managed scope, and keeps your intent (manifest) separate from exact resolved facts (lockfile) — much like `apt`, `brew`, or `npm`, but designed for Minecraft.

## Why Lucy?

Most Minecraft server tools assume a clean install. Lucy works the other way around: it starts from the server you already have, records what it finds, and then lets you decide what it should manage.

## Get started

<Steps>
  <Step title="Install Lucy">
    ```bash
    go install github.com/mclucy/lucy@latest
    ```
  </Step>
  <Step title="Initialize your server">
    Navigate to your server directory and run:
    ```bash
    lucy init
    ```
    Lucy inspects the directory, records what it finds, and creates `.lucy/config.toml`, `.lucy/manifest.json`, and `.lucy/lock.json`.
  </Step>
  <Step title="Add packages">
    ```bash
    lucy add fabric/fabric-api
    lucy add neoforge/create-aeronautics@latest
    ```
    Dependencies are resolved and recorded automatically.
  </Step>
  <Step title="Sync your server">
    ```bash
    lucy install
    ```
    Applies the lockfile to your managed scope.
  </Step>
</Steps>

<Note>
  Lucy is currently under active development. Avoid using it in production environments before the first beta release.
</Note>
